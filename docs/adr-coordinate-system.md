ADR: Building の座標系と配置ルール（原点と基準位置）

コンテキスト:
- 建物（Building）のデータに `x`, `y`, `size` が追加された。
- 配置仕様として「building の配置位置は x, y を基準」「左下が x, y になるように」「x=0, y=0 が画面の中央」を満たす必要がある。
- Next.js(App Router) + MUI 上で `absolute` 配置（CSS は左上原点）を用いて描画しているため、座標系の変換が必要。

要求/目的:
- Firestore の建物座標（論理座標系）を UI の CSS 座標（ピクセル）へ正しく変換する。
- 原点はステージ（描画領域）の中心とし、`(x, y)` は画像の左下が一致するように表示する。
- 既存の演出（ホバー時のグロー）やテーマ構成は維持する。

決定:
- ステージ（描画領域）は `stageW=960`, `stageH=640` の固定サイズとする（将来可変に拡張可能）。
- 画面中心（論理原点）を `cx = stageW/2`, `cy = stageH/2` とする。
- Building 画像は `size` の正方形とし、UI 座標への変換式は以下。
  - CSS left: `left = round(cx + x)`
  - CSS top:  `top  = round(cy - y - size)`
  - 理由: CSS は左上原点だが、UI 上で `(x, y)` は画像左下点を意味するため、高さ `size` を引いて調整する。
- `category === 'center'` は既存同様にホバー時の黄色グローを適用し、それ以外は赤のベースグローを適用する。

実装:
- `src/app/page.tsx` を修正し、リング配置ロジックを廃止。Firestore から取得した全 Building を座標変換して `BuildingItem` で描画。
- ステージを `position: 'relative'` の Box として中央寄せに配置。

影響範囲:
- 表示位置の算出のみ変更。データ取得やコンポーネント API は維持。
- `BuildingItem` の API（left/top/size）をそのまま利用可能。

代替案:
- 原点を左上に置く: 直感に反し、論理モデルとズレるため不採用。
- 右手座標系で y を上向きに保つために `transform` を使って反転: 実装複雑化と hover/filter の継承に注意が必要なため今回は不採用。
