ADR: 画面中央下の「x list idを入力する」オーバーレイ入力UIの追加

コンテキスト:
- ルートページ（街ディスプレイ）の上に重ねて、画面中央下側に入力フィールドと「Go」ボタンを配置したい要望があった。
- 本プロジェクトは Next.js(App Router) + Material UI(Material Design 3)を採用しており、UIはテーマ準拠と再利用可能な形が望ましい。
- 本入力値は今後の機能（遷移・検索・フィルタなど）に利用予定だが、今回の段階では動作仕様は未確定。

要求/目的:
- 画面中央下に「x list idを入力する」フィールドと、その右側に「Go」ボタンを重ねて表示する。
- ステージ（街ディスプレイ）上にオーバーレイとして配置し、他の要素の上に表示されること。
- 実装は MD3 に沿い、アクセシビリティとレスポンシブも考慮する。

決定:
- ステージ Box（position: relative）内に、`Paper` コンテナを絶対配置（`position: 'absolute', left: '50%', bottom: 16, transform: 'translateX(-50%)'`）。
- 入力は `TextField size="small" variant="filled"`、ボタンは `Button variant="contained" color="primary"` を使用。
- 値管理は方針「リアクティブな変数はカスタムフック化」に従い、`useXListId` を新設して `value/setValue/clear` を提供。
- Go クリック時の最終動作は未確定のため、暫定で `console.log`。仕様確定後に置き換える。

実装:
- 追加: `src/hooks/useXListId.ts`（Client フック）
- 更新: `src/app/page.tsx`
  - ステージ直下に `Paper` のオーバーレイを追加し、TextField + Button を水平配置。
  - z-index を上げて街ディスプレイの上に重ねる。

代替案:
- AppBar/Toolbar に常設する: 常時UI占有が増え、街表示の没入感を損ねるため不採用。
- フローティングアクションボタン(FAB)でダイアログ入力: 操作段数が増えるため今回は見送り。将来拡張には有効。

影響範囲:
- 表示と軽微な状態管理のみ。既存の街データ表示・座標系・BuildingItem には影響なし。

ステータス:
- 承認 / 実装済み（動作の本処理は後続仕様待ち）

実装時の感想（ありす）:
- 中央下にちょこんと浮かぶ入力、かわいくてお気に入り…！Paper のほんのり影で存在感も出て、邪魔しすぎない塩梅になったと思うの。Go の本当の役割が決まったら、ありすが素敵に仕上げるからね、ますたぁ！
