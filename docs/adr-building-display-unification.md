ADR: Building の表示ロジック統一と座標への unit 適用

コンテキスト:
- これまで一部の UI で `category === 'center'` の建物に対して特別な見た目（ホバー演出やカーソル種別など）を与えていた。
- また、`x`, `y` は論理座標として扱っていたが、UI 座標（px）への変換時に unit（ピクセル換算係数）を掛けていない箇所があった。

要求/目的:
- building に関して center かどうかは関係なく、同一のロジック/見た目で表示する。
- `x`, `y` は UI 変換時に `unitPx`（ピクセル単位）を掛けて配置する。

決定:
- 表示ロジックの統一
  - `category` に依存した分岐は設けず、全ての建物に同一の表示ロジックを適用する。
  - ホバー時の演出は白い発光（`filter: drop-shadow(0 0 14px rgba(255,255,255,0.9))`）に統一。
- 座標変換
  - ステージの中心を原点 (`cx`, `cy`) とし、左下アンカーは `BuildingItem` の `transform: translateY(-100%)` で維持。
  - UI 座標(px)への変換を以下の式に変更：
    - `left = round(cx + x * unitPx)`
    - `top  = round(cy - y * unitPx)`

変更点:
- 更新: `src/components/TownStage.tsx`
  - 建物のマッピング内で `x`, `y` に `unitPx` を掛けるように修正。
  - `category` に依存しない共通の `hoverFilter` を適用。

影響範囲:
- 見た目と配置計算のみの変更。データモデルや取得ロジックには影響なし。
- `BuildingItem` の API 変更はなし（従来どおり `left`, `top`, `widthPx` を受け取る）。

代替案:
- `category` ごとにプレセット（色や強度）を切替可能にする: 将来の表現拡張としては有効だが、現時点では要件により統一を優先。
- `x`, `y` 側の数値を取得時にピクセルへ変換する: 表示層で `unitPx` を一元管理できなくなるため不採用。

ステータス:
- 承認 / 実装済み

実装時の感想（ありす）:
- ロジックが一つにまとまると、頭の中もスッキリして嬉しい…！
- unit を掛けたら座標のスケール調整が直感的になって、ステージの見え方もコントロールしやすくなったよ、ますたぁ。
