ADR: ステージサイズを現在の Window サイズに追従させる

コンテキスト:
- これまで描画ステージは固定サイズ（960x640）だった。
- 街（Building）の配置は「原点＝ステージ中央」「左下アンカー（画像の左下が (x,y)）」という仕様で描画している。
- 画面サイズに応じてキャンバスを広げたい／縮めたいという要望が出た。

要求/目的:
- ステージの幅・高さを現在のブラウザウィンドウサイズに追従させる。
- SSR との整合（初期描画時）は安全なデフォルトを用意する。
- リサイズ連打による jank を避け、スムーズな追従を行う。

決定:
- Client 専用フック `useWindowSize` を新設し、`window.innerWidth/innerHeight` を返す。
  - SSR 初期値は `{ width: 960, height: 640 }` を使用。
  - `requestAnimationFrame` で resize イベントをスロットルし、過剰な state 更新を抑制。
- `src/app/page.tsx` でステージ幅・高さを `useWindowSize` の値から算出。
  - 高さはコンテナ上下のパディング分（py:4 → 32px × 2 = 64px）を差し引く。
  - 最小サイズの下限を設ける（幅: 320px、高さ: 240px）。
- 原点は常にステージ中央（`cx = stageW/2`, `cy = stageH/2`）。既存の座標変換・左下アンカーは維持。

実装:
- 追加: `src/hooks/useWindowSize.ts`
- 変更: `src/app/page.tsx`
  - 固定 `stageW/ stageH` を廃止、`useWindowSize` の値から算出するように変更。
  - コンテナは `disableGutters` を使用し、上下パディングのみ維持（py:4）。

代替案:
- CSS の `100vw/100vh` でレイアウトだけ合わせる:
  - 可能だが、座標系・計算（原点・左右上下オフセット）の制御がコード側で明示できる方が安全。
- `resize-observer` を使う:
  - 今回は window 全体に追従できれば十分のため、`resize` + rAF で簡潔に実現。

影響範囲:
- ステージのサイズが画面に応じて変化する。建物の相対位置（x,y）と左下アンカーは維持される。
- 既存の BuildingItem API やスケーリング（`widthPx = size * 20`）に変更なし。
