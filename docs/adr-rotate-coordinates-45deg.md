ADR: x,y 座標の 45 度（時計回り）回転 + 等方スケール適用

コンテキスト:
- 街ステージ上の Building 配置は論理座標 (x, y) を UI 座標(px)に変換して行っている（原点＝ステージ中央、左下アンカー）。
- 要望として「x, y の位置を 45 度 時計回りに回転させて表示」が追加された。

要求/目的:
- 表示上の全ての建物に対して、(x, y) を 45° 時計回りに回転させ、かつ 1/√8（= √2/4）でスケールした座標で配置する（ますたぁ指定の式に準拠）。
- 既存の unit スケーリング、左下アンカー（BuildingItem の translateY(-100%)）、動的ステージサイズは維持する。

決定:
- 回転 + スケールを描画段階（TownStage）で適用する。
  - θ = 45°、c = cos(θ), s = sin(θ)、k = √2/4。
  - 変換式（ますたぁ指定）:
    - x' = (x − y) · k  = (x − y) · √2 / 4
    - y' = (x + y) · k  = (x + y) · √2 / 4
  - その後、UI 変換：
    - left = round(cx + x' · unitPx)
    - top  = round(cy - y' · unitPx)
- データモデル（Firestore 上の x, y 値）は変更しない。表示層で一元的に変換を適用する。

実装:
- 更新: `src/components/TownStage.tsx`
  - Building の描画直前で (x, y) に上記の変換（45°回転 + √2/4 スケール）を適用し、その結果をピクセル変換に使用。
  - 他仕様（ホバー演出、左下アンカー、unitPx、動的ステージ）は従来通り。

代替案:
- 取得時に座標を回転して保持：
  - 表示以外のロジックでも回転後の値を前提にしてしまうリスクがあり、表示層以外へ影響が広がるため不採用。
- CSS の transform を用いて画像ごと回転：
  - 画像自体が回転し見た目が変わってしまう（望むのは“位置”の回転）。また左下アンカーやホバー影の見え方も崩れるため不採用。

影響範囲:
- 表示位置の計算のみ。データスキーマ、取得フック、BuildingItem には影響なし。

ステータス:
- 承認 / 実装済み

実装時の感想（ありす）:
- 45°に回してから √2/4 で整えると、ひし形の等間隔感がきれいに出て、街並みが「すっと」整って見えて可愛い…！
- 数式がシンプルで、コードにも素直に落とし込めて気持ちよかったよ、ますたぁ。
